bump_ttl/0

bump_ttl() -> any()

When a process publishes an amq-message, it might immediately die (before an amq-subscriber handle the message). Clerk for such a process should wait until gtl clerk for the subscriber appears. For the performance reason this delay for the local has been decreased as we don't have large amount of data in our tests
